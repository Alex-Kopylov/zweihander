MEGA ENCRYPTED BACKUP STRATEGY: ~/Documents to MEGA Cloud

=============================================================================
OVERVIEW
=============================================================================

Your 2GB Documents folder will be uploaded to MEGA with zero-knowledge
end-to-end encryption. You'll have complete file verification and recovery
capabilities. Estimated time: 15-45 minutes depending on connection speed.

=============================================================================
STEP 1: PRE-BACKUP SETUP (5 minutes)
=============================================================================

1.1 Verify MEGA CLI is installed and logged in:

    mega-whoami

    Expected output: Your MEGA email address
    If "not logged in" error (exit code 57): Run mega-login your-email@gmail.com

1.2 Check available storage on MEGA:

    mega-df -h

    Your Documents folder is ~2GB, and free MEGA accounts include 20GB.
    You need at least 2.5GB free space (buffer for metadata).

1.3 Create the backup destination folder:

    mega-mkdir -p /Backups/Documents-$(date +%Y%m%d)

    Example: /Backups/Documents-20250501
    This creates a timestamped folder for version tracking.


=============================================================================
STEP 2: UPLOAD YOUR DOCUMENTS (10-40 minutes)
=============================================================================

2.1 THE FASTEST & CLEANEST METHOD - Single bulk upload:

    mega-put -c ~/Documents /Backups/Documents-$(date +%Y%m%d)/

    WHY THIS IS OPTIMAL:
    - Uploads entire folder structure recursively with -c flag
    - Single command handles 2GB with all nested folders
    - Preserves directory hierarchy exactly
    - Faster than individual file uploads
    - Clean transaction: all files uploaded together

    PROGRESS TRACKING:
    In another terminal, monitor upload status:

    watch -n 5 'mega-transfers'

    This shows active uploads every 5 seconds.

2.2 ALTERNATIVE: If you want to verify locally first before uploading:

    # Count files and total size locally
    find ~/Documents -type f | wc -l
    du -sh ~/Documents

    Then proceed with the upload command above.


=============================================================================
STEP 3: VERIFY THE BACKUP (CRITICAL - 5 minutes)
=============================================================================

VERIFICATION PROTOCOL - Run all three checks:

3.1 Quick structure verification (lists remote files):

    mega-ls -l /Backups/Documents-$(date +%Y%m%d)/

    Should show: All top-level folders from your Documents

    Then verify nested structure:

    mega-find /Backups/Documents-$(date +%Y%m%d)/ -l | head -20

    Shows: First 20 files with timestamps (proves nested folders exist)


3.2 File count comparison (most important for integrity):

    LOCAL COUNT:
    find ~/Documents -type f | wc -l

    REMOTE COUNT:
    mega-find /Backups/Documents-$(date +%Y%m%d)/ -type f | wc -l

    MUST BE EQUAL. If different, some files failed to upload.

    Example output should look like:
    Local: 1,247 files
    Remote: 1,247 files ✓


3.3 Total size comparison (validates no truncation):

    LOCAL SIZE:
    du -sh ~/Documents

    REMOTE SIZE:
    mega-ls -l /Backups/Documents-$(date +%Y%m%d)/ | tail -1

    Should be approximately equal (MEGA shows total at bottom of listing)
    Small differences (< 1%) are normal due to metadata.

    Example:
    Local size: 2.1 GB
    Remote size: 2.1 GB ✓


3.4 BONUS: Deep integrity check (compare file lists):

    Generate local inventory:
    find ~/Documents -type f -exec ls -lh {} \; | awk '{print $9, $5}' \
      | sort > ~/Documents-local-inventory.txt

    Generate remote inventory:
    mega-find /Backups/Documents-$(date +%Y%m%d)/ -type f -l \
      | awk '{print $NF, $(NF-1)}' | sort > ~/Documents-remote-inventory.txt

    Compare (should be nearly identical):
    diff ~/Documents-local-inventory.txt ~/Documents-remote-inventory.txt

    If no output: All files verified ✓


=============================================================================
STEP 4: ENCRYPTION & SECURITY VERIFICATION
=============================================================================

MEGA uses zero-knowledge encryption automatically - no extra steps needed.

Verify encryption status:

    mega-export /Backups/Documents-$(date +%Y%m%d)/

    Example output shows your backup is stored encrypted.
    Only your account can decrypt these files (not even MEGA staff).

Check storage usage:

    mega-df -h

    Should show ~2GB consumed from your quota.


=============================================================================
STEP 5: OPTIONAL - CREATE RECOVERY PLAN
=============================================================================

5.1 Test download a sample file (proves recovery works):

    mega-get "/Backups/Documents-$(date +%Y%m%d)/YourDocument.pdf" \
      ~/test-recovery/

    Verify the file:
    ls -lh ~/test-recovery/

    Compare with original:
    diff ~/Documents/YourDocument.pdf ~/test-recovery/YourDocument.pdf

    No output = files identical ✓


5.2 Create a full recovery procedure document:

    Save this command for disaster recovery:

    mega-get /Backups/Documents-$(date +%Y%m%d)/ ~/Documents-recovered/

    Will restore entire backup to ~/Documents-recovered/


5.3 Optional: Create a shareable link (if you want backup access elsewhere):

    mega-export -a -f /Backups/Documents-$(date +%Y%m%d)/

    Returns a link like: https://mega.nz/folder/...
    Keep this safe - anyone with it can download your backup.


=============================================================================
COMPLETE ONE-LINER BACKUP SCRIPT
=============================================================================

Copy and paste this to run the entire backup with verification:

#!/bin/bash
set -e  # Exit on any error

BACKUP_NAME="/Backups/Documents-$(date +%Y%m%d)"
LOCAL_PATH="$HOME/Documents"

echo "Starting backup of $LOCAL_PATH to $BACKUP_NAME..."

# Step 1: Create folder
mega-mkdir -p "$BACKUP_NAME"

# Step 2: Upload
echo "Uploading files (this may take 15-40 minutes for 2GB)..."
mega-put -c "$LOCAL_PATH" "$BACKUP_NAME/"

# Step 3: Verify
echo "Verifying backup..."
LOCAL_FILES=$(find "$LOCAL_PATH" -type f | wc -l)
REMOTE_FILES=$(mega-find "$BACKUP_NAME/" -type f | wc -l)

echo "Local files: $LOCAL_FILES"
echo "Remote files: $REMOTE_FILES"

if [ "$LOCAL_FILES" -eq "$REMOTE_FILES" ]; then
    echo "Backup SUCCESSFUL! All $LOCAL_FILES files verified."
else
    echo "ERROR: File count mismatch! Check upload."
    exit 1
fi

echo "Backup complete and verified."


=============================================================================
TROUBLESHOOTING
=============================================================================

Issue: "mega-put: not found"
Fix: Install MEGA CLI: brew install megacmd (or visit mega.io/cmd)

Issue: "File count doesn't match"
Action: Wait 5 minutes (MEGA may still be processing), then re-verify.
        If still different, re-run the upload - partial uploads resume.

Issue: "Storage full"
Action: mega-df -h to check quota. Delete old backups:
        mega-rm -r -f /Backups/Documents-old-date/

Issue: Upload seems stuck
Action: mega-transfers shows status. Safe to interrupt with Ctrl+C
        and resume - MEGA auto-resumes incomplete uploads.

Issue: Connection drops during upload
Fix: MEGA auto-resumes. Just run mega-put again - it picks up where it left off.
