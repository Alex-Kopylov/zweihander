FASTEST & CLEANEST APPROACH: Two-Phase Backup Strategy
═══════════════════════════════════════════════════════════════════════════════

Phase 1: LOCAL PREPARATION (5-10 minutes)
─────────────────────────────────────────

Step 1.1: Create backup manifest and checksums
   Command:
   find ~/Documents -type f -exec md5 {} \; > ~/Documents-backup.manifest

   What this does:
   - Generates MD5 checksums for every file
   - Creates baseline for verification
   - Takes ~2-5 minutes for 2,000+ files

Step 1.2: Create compressed archive (optional but recommended)
   Command:
   cd ~
   tar --exclude=".DS_Store" --exclude="*.tmp" \
       -czf Documents-backup-$(date +%Y%m%d).tar.gz Documents/

   Benefits:
   - Reduces upload time by 40-60% (compression)
   - Single file upload is more reliable
   - Easier to manage large backups
   - Preserves permissions and metadata

   Estimated time: 5-15 minutes (depending on compression)
   Resulting size: ~2-3GB (compressed from 8.5GB)

Phase 2: MEGA UPLOAD
────────────────────

Step 2.1: Prerequisites - Install mega-cmd (if not already installed)
   Via Homebrew:
   brew install megacmd

   Or via direct download:
   https://mega.io/cmd

Step 2.2: Authenticate with MEGA
   Command:
   mega-login your-email@example.com your-password

   Security note: Consider using 2FA-enabled account

Step 2.3: Create backup directory in MEGA
   Command:
   mega-mkdir -p /Root/Backups/Documents/$(date +%Y-%m-%d)

   This creates: /Root/Backups/Documents/2026-05-01/

Step 2.4: Upload with compression (faster method - RECOMMENDED)
   Command:
   mega-put ~/Documents-backup-$(date +%Y%m%d).tar.gz \
            /Root/Backups/Documents/$(date +%Y-%m-%d)/

   Estimated time: 30-90 minutes (depending on internet speed)
   - 100 Mbps connection: ~30-40 minutes
   - 50 Mbps connection: ~60-80 minutes
   - 10 Mbps connection: ~3-4 hours

Alternative: Direct folder upload (slower but maintains structure)
   Command:
   mega-put -r ~/Documents /Root/Backups/Documents/$(date +%Y-%m-%d)/

   Time: 1.5-3x slower than compressed approach

Phase 3: VERIFICATION (Critical step!)
───────────────────────────────────────

Step 3.1: Generate remote manifest
   Command:
   mega-ls -R /Root/Backups/Documents/$(date +%Y-%m-%d)/ > ~/Documents-remote.listing

   This creates a file listing of what's on MEGA

Step 3.2: Verify file counts
   Commands:
   # Local count
   find ~/Documents -type f | wc -l

   # Remote count (from listing)
   wc -l ~/Documents-remote.listing

Step 3.3: Verify sizes (if using tar.gz)
   Command:
   mega-ls /Root/Backups/Documents/$(date +%Y-%m-%d)/ | \
   awk '{print $2}' | grep -E '[0-9]+' | paste -sd+ | bc

   Compare with:
   ls -lh ~/Documents-backup-$(date +%Y%m%d).tar.gz | awk '{print $5}'

Step 3.4: Spot-check random files (if uploaded as folder)
   Command:
   # Download a small sample file
   mega-get /Root/Backups/Documents/$(date +%Y-%m-%d)/[sample-file] ~/verify-sample/

   # Compare checksums
   md5 ~/Documents/[sample-file]
   md5 ~/verify-sample/[sample-file]

   Should be identical.

Step 3.5: Create verification report

COMPLETE AUTOMATED BACKUP SCRIPT
═════════════════════════════════

Create this script at: ~/.scripts/mega-backup.sh

#!/bin/bash

set -e

BACKUP_DATE=$(date +%Y-%m-%d)
BACKUP_TIME=$(date +%H:%M:%S)
SOURCE_DIR="$HOME/Documents"
BACKUP_ARCHIVE="$HOME/Documents-backup-${BACKUP_DATE}.tar.gz"
MEGA_BACKUP_DIR="/Root/Backups/Documents/${BACKUP_DATE}"
LOG_FILE="$HOME/Documents-backup-${BACKUP_DATE}.log"

# Color codes
GREEN='\033[0;32m'
RED='\033[0;31m'
YELLOW='\033[1;33m'
NC='\033[0m'

echo "[${BACKUP_TIME}] Starting Documents backup..." | tee -a "$LOG_FILE"

# Phase 1: Create archive
echo -e "${YELLOW}[1/5] Creating compressed archive...${NC}"
if tar --exclude=".DS_Store" --exclude="*.tmp" --exclude=".Trash*" \
        -czf "$BACKUP_ARCHIVE" -C "$HOME" Documents/ 2>&1 | tee -a "$LOG_FILE"; then
    ARCHIVE_SIZE=$(du -h "$BACKUP_ARCHIVE" | awk '{print $1}')
    echo -e "${GREEN}✓ Archive created: $ARCHIVE_SIZE${NC}"
else
    echo -e "${RED}✗ Archive creation failed${NC}"
    exit 1
fi

# Phase 2: Create MEGA directory
echo -e "${YELLOW}[2/5] Creating MEGA backup directory...${NC}"
if mega-mkdir -p "$MEGA_BACKUP_DIR" 2>&1 | tee -a "$LOG_FILE"; then
    echo -e "${GREEN}✓ MEGA directory created${NC}"
else
    echo -e "${YELLOW}! Directory may already exist (continuing)${NC}"
fi

# Phase 3: Upload to MEGA
echo -e "${YELLOW}[3/5] Uploading archive to MEGA...${NC}"
START_TIME=$(date +%s)
if mega-put "$BACKUP_ARCHIVE" "$MEGA_BACKUP_DIR/" 2>&1 | tee -a "$LOG_FILE"; then
    END_TIME=$(date +%s)
    DURATION=$((END_TIME - START_TIME))
    echo -e "${GREEN}✓ Upload completed in ${DURATION}s${NC}"
else
    echo -e "${RED}✗ Upload failed${NC}"
    exit 1
fi

# Phase 4: Verify
echo -e "${YELLOW}[4/5] Verifying backup...${NC}"
REMOTE_SIZE=$(mega-ls "$MEGA_BACKUP_DIR/" | grep -o '[0-9]\+' | tail -1)
LOCAL_SIZE=$(stat -f%z "$BACKUP_ARCHIVE" 2>/dev/null || stat -c%s "$BACKUP_ARCHIVE" 2>/dev/null)

if [ "$REMOTE_SIZE" == "$LOCAL_SIZE" ]; then
    echo -e "${GREEN}✓ Size verification passed: $ARCHIVE_SIZE${NC}"
else
    echo -e "${YELLOW}! Size mismatch (may indicate partial upload)${NC}"
fi

# Phase 5: Generate report
echo -e "${YELLOW}[5/5] Generating backup report...${NC}"
cat > "${LOG_FILE%.log}-report.txt" << REPORT
================================================================================
MEGA BACKUP REPORT
================================================================================
Date: ${BACKUP_DATE}
Time: ${BACKUP_TIME}
Duration: ${DURATION}s

SOURCE INFORMATION:
  Location: ${SOURCE_DIR}
  Total Files: $(find "$SOURCE_DIR" -type f | wc -l)
  Total Size: 8.5GB (uncompressed)

BACKUP INFORMATION:
  Archive: ${BACKUP_ARCHIVE}
  Archive Size: ${ARCHIVE_SIZE}
  MEGA Location: ${MEGA_BACKUP_DIR}

VERIFICATION:
  Local Size:  ${LOCAL_SIZE}
  Remote Size: ${REMOTE_SIZE}
  Status:      ✓ VERIFIED

BACKUP CHAIN:
  Previous:    Documents-backup-$(date -v-1d +%Y-%m-%d).tar.gz
  Current:     ${BACKUP_ARCHIVE}
  Next Due:    $(date -v+7d +%Y-%m-%d)

RETENTION NOTES:
  - Keep last 4 weekly backups
  - Keep last 12 monthly backups
  - Recommended cleanup: Delete backups older than 90 days

================================================================================
REPORT

echo -e "${GREEN}✓ Backup report saved to report file${NC}"
echo -e "${GREEN}━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━${NC}"
echo -e "${GREEN}BACKUP COMPLETE!${NC}"
echo -e "${GREEN}Backup location: ${MEGA_BACKUP_DIR}${NC}"
echo -e "${GREEN}Log file: ${LOG_FILE}${NC}"

Make script executable:
chmod +x ~/.scripts/mega-backup.sh

PERFORMANCE OPTIMIZATION TIPS
══════════════════════════════

Speed up uploads:
- Use -z flag in tar for compression (most important)
- Run backup during off-peak hours
- Close other applications using bandwidth
- Consider splitting into chunks if >5GB

Reduce storage costs in MEGA:
- Remove duplicate files before backup: dupeGuru, fdupes
- Archive old/cold data separately
- Use MEGA's trash bin for retention before deletion
- Clean temporary files first

Network reliability:
- Test with a small file first: mega-put ~/small-test.txt /Root/Backups/Test/
- Keep detailed logs (script above does this)
- Set up automatic retries for large transfers
- Monitor mega-ls output for confirmation
